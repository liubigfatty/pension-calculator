## 二、技术选型：推荐「云调用支付API」

### 为什么选这个方案

| 方案 | 复杂度 | 需管理证书 | 需配置APIv3密钥 | 推荐 |
|:----|:------:|:----------:|:--------------:|:----:|
| **A. 云调用支付API** ⭐ | 低 | ❌ 不需要 | ❌ 不需要 | **✅** |
| B. 直接调用APIv3（RSA签名） | 高 | ✅ 需要上传证书 | ✅ 需要 | ❌ |
| C. 第三方SDK（wechatpay-node-sdk） | 中 | ✅ 需要上传证书 | ✅ 需要 | ❌ |

**理由**：云函数已经依赖了 `wx-server-sdk ~2.6.3`，直接使用 `cloud.pay.unifiedOrder()` 即可，不需要管理商户证书、不需要处理 APIv3 签名。云开发平台自动处理了和微信支付服务器之间的安全通信。

### 云调用支付API的限制
- 仅限云函数中使用（我们已经用云函数了 ✅）
- 需要在小程序管理后台「云开发」中配置支付能力
- 订单金额单位是**分**（1元=100分）

## 三、具体实施步骤

### 第1步：配置微信支付商户平台（你做）

> ⏱ 大约需要 10-15 分钟

**1.1 获取APIv3密钥**
- 登录 [微信支付商户平台](https://pay.weixin.qq.com/)
- 进入「账户中心 → API安全 → 设置APIv3密钥」
- 设置一个32位随机字符串（比如：`qwertyuiop1234567890asdfghjklzxcv`）
- **记下来，后续要用**

> ⚠️ 对于云调用方案，这个APIv3密钥只需要配置在云开发环境变量中，不需要在代码里管理证书。

**1.2 配置回调通知地址（可选，强化验证用）**
- 微信支付商户平台 → 产品中心 → 开发配置 → JSAPI支付配置
- 设置支付回调通知域名（即云函数HTTP触发地址的域名部分）

**1.3 获取商户证书序列号（云调用方案不需要）**
- 云调用方案不需要商户证书，跳过此步

**1.4 关联小程序AppID**
- 登录 [微信小程序后台](https://mp.weixin.qq.com/)
- 功能 → 微信支付 → 查看已关联的商户号
- 确认 `wx76075ba352d5333c`（小程序AppID）和 `1747046083`（商户号）已关联
- 如未关联，点击「关联商户号」输入商户号

**1.5 云开发环境配置**
- 登录 [微信小程序后台](https://mp.weixin.qq.com/)
- 云开发 → 选择环境 → 设置 → 添加环境变量
  - 键名：`APIV3_KEY`
  - 键值：你设置的32位APIv3密钥
- 键名：`MCHID`
  - 键值：`1747046083`
- 键名：`APPID`
  - 键值：`wx76075ba352d5333c`

### 第2步：创建支付云函数 CreateOrder（我来写）

**2.1 云函数代码重写**

使用 `cloud.pay.unifiedOrder` 替代手写 APIv3 签名：

```javascript
// cloud/functions/createOrder/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

/**
 * 云调用支付 — 统一下单
 * 文档：https://developers.weixin.qq.com/miniprogram/dev/wxcloud/guide/payment/cloud-pay.html
 */
exports.main = async (event) => {
  const { total_fee, openid } = event
  if (!openid) return { code: -1, msg: '缺少openid' }

  try {
    const outTradeNo = 'RP' + Date.now() + Math.random().toString(36).slice(-4).toUpperCase()

    const result = await cloud.pay.unifiedOrder({
      body: '养老金测算深度报告',   // 商品描述
      outTradeNo,                   // 商户订单号
      spbillCreateIp: '127.0.0.1', // 终端IP
      subMchId: process.env.MCHID, // 商户号
      totalFee: total_fee || 100,  // 金额：分（100 = 1元）
      envId: process.env.ENV_ID,   // 云环境ID（需设置环境变量）
      functionName: 'payCallback', // 支付回调云函数
    })

    if (result && result.prepayId) {
      // 直接返回 prepay_id，前端用它调起支付
      return { code: 0, data: { prepayId: result.prepayId, outTradeNo } }
    }
    return { code: -1, msg: JSON.stringify(result) }
  } catch (e) {
    return { code: -1, msg: e.message }
  }
}
```

### 第3步：前端支付流程（我来写）

**3.1 result.js — goReport() 改为支付流程**

```javascript
// pages/result/result.js
goReport() {
  const app = getApp()
  const openid = app.globalData.openid || wx.getStorageSync('openid')

  if (!openid) {
    // 先登录获取 openid
    wx.showLoading({ title: '获取用户信息...' })
    wx.login({
      success: (res) => {
        // 用云函数获取 openid（或直接从缓存获取）
        const cachedOpenid = wx.getStorageSync('openid')
        if (cachedOpenid) {
          this._startPayment(cachedOpenid)
        } else {
          wx.hideLoading()
          wx.showToast({ title: '登录失败，请重试', icon: 'none' })
        }
      }
    })
    return
  }
  this._startPayment(openid)
},

_startPayment(openid) {
  wx.showLoading({ title: '创建订单...', mask: true })

  wx.cloud.callFunction({
    name: 'createOrder',
    data: { total_fee: 100, openid }  // 100分 = 1元
  }).then(res => {
    wx.hideLoading()
    if (res.result.code !== 0) {
      wx.showToast({ title: res.result.msg || '下单失败', icon: 'none' })
      return
    }

    const { prepayId, outTradeNo } = res.result.data

    // 如果有服务器端生成的支付参数，使用 wx.requestPayment
    // 如果用 cloud.pay.unifiedOrder，需要调用另一个接口获取支付参数
    // 实际上 cloud.pay.unifiedOrder 返回后，还需要调 cloud.pay.getSign 来生成前端调起支付的参数

    wx.showLoading({ title: '正在调起支付...', mask: true })

    // 获取支付签名参数
    wx.cloud.callFunction({
      name: 'getPaymentParams',
      data: { prepayId, outTradeNo }
    }).then(payRes => {
      wx.hideLoading()
      if (payRes.result.code !== 0) {
        wx.showToast({ title: '获取支付参数失败', icon: 'none' })
        return
      }

      const params = payRes.result.data
      wx.requestPayment({
        ...params,
        success: (payResult) => {
          wx.showToast({ title: '支付成功', icon: 'success' })
          // 支付成功 → 标记已购买并跳转报告页
          wx.setStorageSync('report_paid', '1')
          wx.navigateTo({ url: '/pages/report/report' })
        },
        fail: (err) => {
          if (err.errMsg.includes('cancel')) {
            wx.showToast({ title: '已取消支付', icon: 'none' })
          } else {
            wx.showToast({ title: '支付失败', icon: 'none' })
          }
        }
      })
    })
  }).catch(err => {
    wx.hideLoading()
    wx.showToast({ title: '网络错误', icon: 'none' })
  })
}
```

**3.2 新增 getPaymentParams 云函数**

`cloud.pay.unifiedOrder` 返回 `prepayId` 后，还需要用 `cloud.pay.getSign` 生成前端调起支付所需的签名参数：

```javascript
// cloud/functions/getPaymentParams/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event) => {
  const { prepayId, outTradeNo } = event
  if (!prepayId) return { code: -1, msg: '缺少prepayId' }

  try {
    const params = await cloud.pay.getSign({
      prepayId,
      signType: 'MD5',  // 或用 'HMAC-SHA256'
    })
    return { code: 0, data: params }
  } catch (e) {
    return { code: -1, msg: e.message }
  }
}
```

**3.3 report.js — 支付护城河**

```javascript
// pages/report/report.js — onLoad 检查是否已支付
onLoad(options) {
  const isPaid = wx.getStorageSync('report_paid') === '1'
  if (!isPaid) {
    wx.showModal({
      title: '需要先支付',
      content: '完整报告需支付0.99元，是否前往支付？',
      success: (res) => {
        if (res.confirm) {
          wx.navigateBack()  // 返回结果页重新支付
        } else {
          wx.navigateBack()
        }
      }
    })
    return
  }
  // ... 正常加载报告内容
}
```

### 第4步：支付回调处理（我来写）

**4.1 payCallback 云函数**

支付成功后，微信支付服务器会回调通知结果。这个回调可以用来：
- 更新订单状态
- 发送订阅消息提醒用户

```javascript
// cloud/functions/payCallback/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event) => {
  // event 包含微信支付回调的完整参数
  const { return_code, out_trade_no, transaction_id, total_fee, openid } = event

  if (return_code === 'SUCCESS') {
    console.log('✅ 支付成功:', out_trade_no, transaction_id, total_fee)

    // 后续可扩展：
    // - 记录支付订单到数据库
    // - 发送订阅消息

    return { code: 'SUCCESS', message: 'ok' }
  }

  console.log('❌ 支付失败:', event)
  return { code: 'FAIL', message: '支付失败' }
```

## 七、风险点与预案

| 风险 | 影响 | 预案 |
|:----|:----|:-----|
| `cloud.pay.unifiedOrder` 需开通云调用支付能力 | 无法调用 | 在小程序后台申请开通，通常即时生效 |
| 微信审核不通过 | 无法上线 | 审核备注说明「支付功能使用云开发官方API，安全可靠」 |
| 用户支付成功后未跳转 | 用户体验差 | report.js 的 onLoad 同时检查缓存+云数据库中的支付状态 |
| 回调未及时收到 | 订单状态不一致 | 前端以 wx.requestPayment 的 success 为准，回调做二次确认 |
