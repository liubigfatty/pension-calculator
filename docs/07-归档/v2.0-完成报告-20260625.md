- ✅ `createOrder` 云函数骨架已创建

### 5.2 支付集成的关键问题

#### ❌ 问题一：前端支付流程未打通
`result.js` 的 `goReport()` 现在是**直接跳转报告页**，没有调用支付：
```javascript
// 当前代码（无支付）
goReport() {
  wx.navigateTo({ url: '/pages/report/report' })
}
```
需要改为：调用云函数 `createOrder` → 调起 `wx.requestPayment` → 支付成功后才跳转。

#### ❌ 问题二：APIv3 签名逻辑需要验证
`createOrder/index.js` 使用 `crypto.createHmac('sha256', APIv3)` 签名，但微信支付 APIv3 的 HTTP Authorization 头需要使用**商户证书的 RSA-SHA256 签名**，而非单纯的 HMAC。需要：
1. 在云函数中配置商户证书序列号和私钥
2. 使用正确的 RSA-SHA256 签名算法
3. APIv3 密钥（32位）仅用于回调通知和解密，不用于请求签名

#### ❌ 问题三：notify_url 占位符未替换
`createOrder/index.js` 第46行：
```javascript
notify_url: 'https://YOUR_CLOUD_ID.service.tcloudbase.com/payCallback'
```
需要替换为实际的云函数 HTTP 触发器地址。

#### ❌ 问题四：APIv3 密钥未配置
代码中 `APIv3_KEY` 从环境变量读取，但目前未设置。需要在云函数控制台配置。

### 5.3 支付集成建议
如果现在提交审核（v1.0.5 已在审核中），建议：
1. **v1.0.5 保持审核状态不变**（免费版，不含支付）
2. **审核通过后发布 v1.0.5** 作为基础版本上线
3. **v2.0 作为下一迭代**，完成支付集成后再提审

如果直接上传开发版本（跳过审核）测试支付，需要注意：
- 支付功能需要在微信小程序后台配置「支付能力」
- 需要在「小程序后台 → 功能 → 微信支付」中关联商户号
- 建议先在开发工具中真机测试支付流程，再正式提审
