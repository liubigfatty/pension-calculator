## 一、快速开始（3步添加新省份）

### 步骤1：创建省份模块文件

在 `data/provinces/` 目录下创建 `xx省.js`（如 `hunan.js`）。

**最小模板**（无视同缴费、无特殊增发）：

```javascript
// data/provinces/hunan.js
const PROV_TAG = 'hunan'

// 计发基数（元/月，官方数据）
const PROV_BASE = {
  // 2024年及以前：官方公布的各年度计发基数
  2023: 7417,  // 来源：岳阳市政府网站信件回复（2024-12）
  2024: 7618,  // 来源：岳阳市政府网站信件回复（2025-01）
  2025: 7694,  // 来源：湘人社规〔2025〕30号 | URL: rst.hunan.gov.cn/...
  2026: 7694,  // 预估（待官方公布）
}

// 全省退休人员人均养老金（用于独生子女补贴）
const AVG_PENSION_ER_PENSION = {
  2023: 3293,  // 来源：云人社发〔2023〕41号（云南参考）
  // ... 待补充湖南省数据
}

// 城市列表（无地级市区分则留空数组）
const CITY_LIST = []

// 模块配置（需要哪些就加哪些）
const MODULES = ['base', 'personal', 'transition']  // 无增发、无其它加发

// 测试案例（至少2个，来源必须是官方核定表）
const cases = [
  {
    name: '湖南-男-2025退休（待验证）',
    note: '⚠️ 案例数据待替换为官方核定表',
    input: {
      birthYear: 1965, birthMonth: 6,
      workYear: 1985, workMonth: 6,
      cityType: 'prov',
      avgIndex: 0.85,
      personalAccInput: 85000,
      totalYears: 40,
      sightYears: 10.0,
      skipDelay: true,
    },
    expected: {
      base: 2846.78,
      personal: 611.51,
      transition: 850.19,
      other: 0,
      total: 4308.48,
    },
  },
]

// 引擎配置
function getEngineConfig() {
  const modules = {};
  if (MODULES.includes('base'))       modules.basic_pension = { enabled: true, rate_per_year: 0.01 };
  if (MODULES.includes('personal'))  modules.personal_account = { enabled: true };
  if (MODULES.includes('transition')) {
    modules.transitional_pension = { enabled: true };
    const TRANS_COEF = 0.013;  // 过渡系数 1.3%
    if (TRANS_COEF) {
      if (typeof TRANS_COEF === 'number') {
        modules.transitional_pension.coefficient = TRANS_COEF;
      }
    }
  }
  // 如有独生子女补贴：
  // if (MODULES.includes('other')) {
  //   modules.special_addition = {
  //     enabled: true,
  //     type: 'one_child',
  //     rate: 0.05,
  //     avgPensionData: AVG_PENSION_ER_PENSION || {},
  //   };
  // }

  return {
    province: PROV_TAG,
    name: '湖南省',
    base_rates: { prov: PROV_BASE },
    modules: modules,
    account_start: '1995-10',  // 建账时间（待官方文件确认）
    cutoff_date: '1995-12-31',   // 视同缴费cutoff时间
    usePreAccountYears: false,
    cities: CITY_LIST || [],
    cases: cases || [],
    notes: '⚠️ 1995-2022年计发基数为估算值，待补充官方数据。',
  };
}

module.exports = {
  PROV_TAG,
  PROV_BASE,
  CITY_LIST,
  MODULES,
  getEngineConfig,
  cases,
}
```

### 步骤2：更新引擎（如需支持特殊公式）

**什么情况下需要改引擎？**

| 特殊情况 | 需要改的引擎函数 | 示例省份 |
|---------|----------------|---------|
| 过渡性养老金有特殊公式 | `calcTransitionalPension()` | 上海（`xuzhang`）、北京（`beijing`）、江苏（`jiangsu`）、四川（`sichuan`） |
| 有独生子女补贴 | `calcSpecialAddition()` 新增 `type` | 云南（`one_child`） |
| 有增发基础养老金 | `calcExtraPension()` | 吉林 |
| 有调节金 | `calcAdjustmentFund()` | 甘肃 |

**改引擎的标准流程**：

1. 在对应函数里新增 `formula_type` 分支（如 `if (mod.formula_type === "hunan")`)
2. 在 `parseInput()` 里新增参数解析（如需新参数）
3. 在 `calculate()` 里调用时传递新参数
4. 在该省的 `getEngineConfig()` 里配置 `formula_type`

### 步骤3：运行测试

```bash
node test_all_provinces.js
```

- 测试脚本会**自动发现** `data/provinces/*.js`，无需手动维护省份列表
- 如果案例 `expected` 全为0，会显示"⚠️ 案例数据待验证"，不影响其他省份测试

## 二、必填参数清单

### 2.1 计发基数（`PROV_BASE`）

**必须收集**：各年度养老金计发基数（元/月）

**去哪里找**：省人社厅官网 → 搜索"关于公布XXXX年度养老金计发基数"

**记录格式**：

```javascript
const PROV_BASE = {
  2023: 7417,  // 年平均XX元 ÷ 12 | 来源：湘人社规〔2023〕XX号 | URL: ...
  2024: 7618,  // 来源：湘人社规〔2024〕XX号 | URL: ...
  2025: 7694,  // 来源：湘人社规〔2025〕30号 | URL: rst.hunan.gov.cn/...
  2026: 7694,  // 预估（待官方公布）
}
```

**双基数省份特殊处理**（如云南、甘肃）：

- `base_rates.prov` = 计发基数（小口径，在岗职工平均工资）
- `avg_salary_history` = 缴费基数（全口径，城镇单位就业人员加权平均）

### 2.2 过渡系数

**固定值**：直接写在 `getEngineConfig()` 里

```javascript
modules.transitional_pension = {
  enabled: true,
  coefficient: 0.013,  // 1.3%
}
```

**分段规则**（如吉林）：需要在引擎里写判断逻辑

```javascript
modules.transitional_pension = {
  enabled: true,
  coefficient_over_20: 0.014,   // 实际缴费>20年
  coefficient_under_20: 0.012,  // 实际缴费≤20年
}
```

**特殊公式**（如上海、北京、江苏、四川）：需要改引擎

```javascript
modules.transitional_pension = {
  enabled: true,
  formula_type: "shanghai",  // 引擎会根据 formula_type 选择对应公式
}
```

### 2.3 建账时间（`account_start`）

**作用**：计算"建账前缴费年限"（用于过渡性养老金计算）

**去哪里找**：省人社厅官网 → 搜索"个人账户制度建立"

**配置方式**：

```javascript
// 方式1：在 getEngineConfig() 里返回
function getEngineConfig() {
  return {
    // ...
    account_start: '1995-10',  // 建账时间
  }
}

// 方式2：在测试案例 input 里手动指定（用于验证官方核定表）
{
  input: {
    // ...
    accountStart: '2003-01',  // 覆盖默认建账时间
  }
}
```

### 2.4 视同缴费 cutoff 时间

**作用**：确定视同缴费年限计算截止时间

**常见值**：
- 大部分省份：1995-12-31（建账前）
- 上海：1992-12-31（特殊）

**配置方式**：目前引擎默认按建账时间计算，无需额外配置。如果省份有特殊cutoff时间，需要在引擎里加判断。

## 三、特殊规则实现指南

### 3.1 独生子女补贴（`one_child`）

**适用省份**：云南（其他省份如有，按相同方式实现）

**引擎实现**（已在 `calcSpecialAddition()` 里）：

```javascript
} else if (mod.type === 'one_child') {
  // 独生子女补贴：全省退休人员人均养老金 × 比例
  const retireYear = params?.context?.retireYear || 2025;
  const avgPensionData = mod.avgPensionData || {};
  const avgPension = avgPensionData[retireYear] || avgPensionData[2023] || 0;
  const rate = mod.rate || 0.05;
  const amount = Math.round(avgPension * rate * 100) / 100;
  return {
    amount,
    description: `独生子女补贴：全省人均养老金${avgPension}元(${retireYear}年) × ${(rate * 100)}% = ${amount.toFixed(2)}元`
  }
}
```

**省份模块配置**（如 `yunnan.js`）：

```javascript
if (MODULES.includes('other')) {
  modules.special_addition = {
    enabled: true,
    type: 'one_child',
    rate: 0.05,  // 5%
    avgPensionData: AVG_PENSION_ER_PENSION || {},
  };
}
```

**数据准备**：需要收集该省"全省退休人员人均养老金"各年度数据

### 3.2 虚账实记（`xuzhang`，上海特有）

**适用省份**：上海

**引擎实现**（已在 `calcTransitionalPension()` 里）：

```javascript
if (mod.formula_type === "shanghai") {
  const retireBase = params?.retireBase || provBase
  const sight = params?.sightYears || effectiveYears
  const coefSH = mod.coefficient || 0.012
  const xuzhang = params?.xuzhang || 0

  const basePart = retireBase * sight * 1.0 * coefSH
  const xuzhangPart = xuzhang > 0 ? xuzhang / 120 : 0
  const amount = Math.round((basePart + xuzhangPart) * 100) / 100

  let desc = '上海过渡性养老金: ' + retireBase.toFixed(2) + ' × 1.0 × ' + sight.toFixed(2) + '年 × 1.2%'
  if (xuzhang > 0) {
    desc += ' + ' + xuzhang.toFixed(2) + ' ÷ 120'
  }
  desc += ' = ' + amount.toFixed(2) + '元'
  return { amount, description: desc }
}
```

**省份模块配置**（如 `shanghai.js`）：

```javascript
if (MODULES.includes('transition')) {
  modules.transitional_pension = { enabled: true, formula_type: "shanghai" };
}
```

**`parseInput()` 里新增 `xuzhang` 解析**：

```javascript
// 上海特殊参数：虚账实记总额（用于过渡性养老金计算）
const xuzhang = parseFloat(inputData.xuzhang) || null
```

**`calculate()` 里调用时传递 `xuzhang`**：

```javascript
const transPension = calcTransitionalPension({
  // ... 其他参数
  xuzhang: data.xuzhang
})
```

### 3.3 增发基础养老金（吉林特有）

**适用省份**：吉林

**引擎实现**：`calcExtraPension()` 函数

**规则**：实际缴费>20年部分分段增发
- 21-25年：0.15%
- 26-30年：0.20%
- 31年及以上：0.25%

**省份模块配置**（如 `jilin.js`）：

```javascript
if (MODULES.includes('extra')) {
  modules.extra_pension = { enabled: true }
}
```

## 五、测试案例规范

### 5.1 案例来源要求

**⚠️ 核心原则：测试案例必须来自官方核定表，不能AI自己编！**

**正确做法**：
1. 用户提供官方核定表截图或PDF
2. 从核定表提取数据，填入 `input` 和 `expected`
3. 在案例 `note` 字段记录来源（如 `⚠️ 案例来源：用户2026-06-09提供 | 长沙市XX区社保局核定表`）

**错误做法**：
- ❌ 用引擎计算值作为期望值（这是"用引擎验证引擎"，无意义）
- ❌ 从第三方网站复制案例（数据准确性无法保证）

### 5.2 案例格式

```javascript
{
  name: '湖南-男-2025退休（正式核定·60岁）',  // 案例名称（用于测试输出）
  note: '⚠️ 案例来源：用户2026-XX-XX提供 | XX市社保局核定表',
  officialDocument: '湘社保核〔2025〕XXXX号',  // 可选：核定表编号
  input: {
    birthYear: 1965, birthMonth: 6,
    workYear: 1985, workMonth: 6,
    cityType: 'prov',        // 'prov' = 全省，'cityname' = 具体城市
    avgIndex: 0.8183,        // 平均缴费指数
    personalAccInput: 60000, // 个人账户余额
    totalYears: 40,          // 累计缴费年限
    sightYears: 11.83,       // 视同缴费年限
    skipDelay: true,         // 跳过延迟退休计算（验证场景用）
    // 可选参数：
    // months: 139,           // 计发月数（手动指定，用于验证官方核定表）
    // accountStart: '1995-10', // 建账时间（手动指定，覆盖默认）
    // xuzhang: 60157.46,    // 虚账实记总额（上海）
  },
  expected: {
    base: 3097.15,           // 基础养老金
    personal: 1321.07,       // 个人账户养老金
    transition: 1653.34,     // 过渡性养老金
    extra: 0,                // 增发基础养老金（吉林）
    other: 0,                // 其它加发（独生子女、调节金等）
    total: 6071.56,          // 总养老金
  },
}
```

### 5.3 临时通过策略

**如果暂时没有官方核定表**，可以先创建案例框架，`expected` 全填0，标注"待验证"：

```javascript
{
  name: '湖南-男-2025退休（待验证）',
  note: '⚠️ 案例数据待替换为官方核定表',
  input: { /* ... */ },
  expected: {
    base: 0,
    personal: 0,
    transition: 0,
    total: 0,
  },
}
```

测试脚本会显示"⚠️ 案例数据待验证"，不影响其他省份测试。

**等用户提供官方核定表后**，再更新 `expected` 为真实值。

## 六、常见错误及解决方案

### 错误1：测试脚本报"无案例数据"

**原因**：`module.exports` 里没有导出 `cases`

**解决**：在省份模块文件末尾，确保导出 `cases`：

```javascript
module.exports = {
  PROV_TAG,
  PROV_BASE,
  CITY_LIST,
  MODULES,
  cases,          // ⚠️ 必须有这个
  getEngineConfig,
}
```

### 错误2：引擎返回 `legal.base` 是 `undefined`

**原因**：引擎返回的字段名是嵌套对象，不是扁平字段

**正确字段名**：

```javascript
// ❌ 错误
result.legal.base
result.legal.personal
result.legal.transition

// ✅ 正确
result.legal.basicPension.amount
result.legal.personalAccount.amount
result.legal.transitionalPension.amount
result.legal.specialAddition.amount   // 独生子女补贴等
result.legal.extraPension.amount      // 增发基础养老金（吉林）
result.legal.total                    // 总养老金
```

### 错误3：过渡性养老金计算错误（差值很大）

**常见原因**：
1. 该省有特殊公式（如上海、北京、江苏、四川），但 `formula_type` 没配置
2. `xuzhang` 参数没传递（上海）
3. 建账时间（`accountStart`）配置错误

**解决**：
1. 检查该省是否有特殊过渡性养老金公式
2. 如有，在 `getEngineConfig()` 里配置 `formula_type`
3. 如需新参数（如 `xuzhang`），在 `parseInput()` 里新增解析

### 错误4：`Edit` 工具找不到匹配字符串

**原因**：文件是 `\r\n` 换行（Windows格式），`Edit` 工具对空格/换行敏感

**解决**：用 Node.js 脚本直接修改文件内容

```javascript
// 示例：修改文件内容
const fs = require('fs')
let content = fs.readFileSync('path/to/file.js', 'utf8')
content = content.replace('旧字符串', '新字符串')
fs.writeFileSync('path/to/file.js', content, 'utf8')
```

## 八、附录：引擎函数清单

| 函数名 | 作用 | 需要改的情况 |
|--------|------|-------------|
| `parseInput()` | 解析输入参数 | 新增参数（如 `xuzhang`） |
| `calcBasePension()` | 计算基础养老金 | 全国统一，无需改动 |
| `calcPersonalPension()` | 计算个人账户养老金 | 全国统一，无需改动 |
| `calcTransitionalPension()` | 计算过渡性养老金 | 该省有特殊公式时 |
| `calcExtraPension()` | 计算增发基础养老金 | 该省有增发时（如吉林） |
| `calcSpecialAddition()` | 计算其它加发 | 该省有独生子女/调节金等时 |
| `calcAdjustmentFund()` | 计算调节金 | 该省有调节金时（如甘肃） |
| `calculate()` | 主计算函数 | 新增参数传递时 |
