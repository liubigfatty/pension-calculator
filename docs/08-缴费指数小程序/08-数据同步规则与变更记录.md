## 一、网页版与小程序的数据同步规则

### 1.1 单一真相源（唯一权威数据源）
- 路径：`cloudfunctions/calculate/provinces/*.js` + 同名 `.json`
- `.js` 提供 `getEngineConfig().avg_salary_history`（逐年社平工资，元/月）
- 同名 `.json` 提供 `verified_count`（该省已人工核对年数，校准标识；**位置不统一**：11 省顶层、8 省嵌套在 `notes` 内、11 省缺失）

### 1.2 生成链（手动串联，无自动 CI）
```
真相源 (provinces/*.js + *.json)
   │  node scripts/_gen_province_data.js
   ▼
index-mini/cloudfunctions/calcIndex/provinces-data.js   ← 小程序云函数数据包
   │  node scripts/build-web-index.js
   ▼
web-index/provinces-index-data.js  +  web-index/calc-index.js   ← 网页版
```
⚠️ **顺序硬依赖**：网页版的数据源是「小程序的 provinces-data.js」，不是真相源本身。
必须先跑 `_gen_province_data.js` 再跑 `build-web-index.js`，顺序反了网页版会滞后于真相源。

### 1.3 改数据后的同步 / 校验命令（必跑）
```bash
node scripts/_gen_province_data.js     # 真相源 → 小程序包（含 verified_count 深度递归透传）
node scripts/build-web-index.js        # 小程序包 + 引擎 → 网页版（含 31 省自检，不符则 exit 1）
node scripts/verify-index-mini.js      # 31 省 正向/反推一致收敛校验
node scripts/verify-web.js             # 网页版引擎抽样对账
```

### 1.4 ⚠️ 已知瑕疵（待修，非阻塞）
`scripts/build-web-index.js` 第 8–9 行注释写「本脚本把同一份真相源包装成浏览器包」，
但代码 `DATA_SRC` 实际指向 `index-mini/.../provinces-data.js`（小程序包）。**注释与代码不符**，易误导。
建议改为「读小程序数据包」。
