# 缴费指数小程序 · P0 修复规格（上线阻断项）

> 范围：3 个 P0（封顶保底缺失 / 数据口径不统一 + verified_count 未透传 / 隐私合规缺失）。
> 本文档**只给修复规格，不修改任何代码或数据文件**。修复由实现方按本规格执行。
> 关联文档：`06-计算规则统一与决策留痕.md`（规则依据）、`02-校准报告.md`（口径决策）。

## P0-1 单年指数封顶保底（clamp）缺失

### 问题
- 三处引擎在 `yearIndex = rec.baseAvg / socialAvg` 后**均未 clamp**，高收入年指数可远超 3.0、异常低基数年可低于 0.6：
  - `index-mini/cloudfunctions/calcIndex/calcIndex.js:144`
  - `index-engine/calcIndex.js:144`
  - `web-index/calc-index.js:147`（由 `index-engine` 经 `scripts/build-web-index.js` 生成，改源后重建即得）
- 违反规则②；`index-mini/AGENTS.md:27` 已写明区间但代码未实现。

### 修复规格
- 在第 144 行（三引擎对应位置）之后插入：
  ```js
  // 规则②：单年指数封顶 3.0 / 保底 0.6（仅影响平均指数，不影响账户入账）
  const yearIndexClamped = Math.min(3.0, Math.max(0.6, yearIndex))
  ```
- 将后续对原始 `yearIndex` 的**加权平均累加**与**逐年明细记录**改用 `yearIndexClamped`：
  - `totalIndexSum += yearIndexClamped * rec.months`（原 `yearIndex`）
  - `yearsDetail` 中该年 `index` 字段记 `yearIndexClamped`（原 `yearIndex`）
- **不动**：账户入账 `annualAccountPay = rec.baseAvg * rec.months * 0.08`（用真实基数，不 clamp）。

### 重建与验证
- 改 `index-mini` 与 `index-engine` 两处源码后执行 `node scripts/build-web-index.js`（自动把 clamp 带入 `web-index/calc-index.js`）。
- 单测：吉林样本某年基数设为社平 4 倍 → 该年指数应 = 3.0；设为社平 0.3 倍 → 应 = 0.6；平均指数落在 [0.6, 3.0]。

## P0-2 社平数据包不统一 + verified_count 未透传

### 问题
- 真相源 `cloudfunctions/calculate/provinces/*.json` 含 `verified_count`，但生成脚本 `scripts/_gen_province_data.js` 只读 `.js` 的 `getEngineConfig().avg_salary_history`（例 `hebei.js:174` 仅返回社平），**不读 verified_count**，导致部署包 `index-mini/.../provinces-data.js`（生成于 2026-07-12）无该字段。
- 部署包与真相源存在年度错位 / 数值漂移（审计：最大 25.7%，如 fujian@2023；hubei 22.6%；sichuan / yunnan@2026 12.7%），需从真相源重生成以消除。

### 修复规格
1. **给生成脚本加 verified_count 透传**——编辑 `scripts/_gen_province_data.js` 的遍历循环（当前约 line 35-51）：在 `require(.js)` 拿到 `cfg` 后，读取同目录同名 `.json` 的 `verified_count`，并入输出：
   ```js
   const jsonPath = path.join(SRC_DIR, slug + '.json')
   let verifiedCount = undefined
   if (fs.existsSync(jsonPath)) {
     try { verifiedCount = JSON.parse(fs.readFileSync(jsonPath, 'utf8')).verified_count } catch (e) {}
   }
   out[slug] = {
     name: cfg.name || slug,
     avg_salary_history: withExtrapolation(avgRaw),
     verified_count: verifiedCount   // 新增：随包下发，用于结果页信心提示
   }
   ```
   （替代方案：让 31 个 `.js` 的 `getEngineConfig()` 直接返回 `verified_count`，但改动面大，不推荐。）
2. **重新生成部署包**（在仓库根执行）：
   ```bash
   node scripts/_gen_province_data.js     # 产出 index-mini/.../provinces-data.js（含 verified_count）
   node scripts/build-web-index.js        # 自动把 verified_count 带入 web-index/provinces-index-data.js（脚本为整文件包裹，字段自动透传）
   ```
3. **如主养老金引擎也消费同源数据**：按需 `node scripts/sync-provinces.js`（用法见文件头 `node scripts/sync-provinces-v2.js`）同步主程序副本。

### 验证
- `node scripts/verify-index-mini.js`：部署包与真相源逐省逐年比对，偏差须 ≤ 容差（建议 ≤1%，外推年除外）；`verified_count` 与 `.json` 一一对应。
- `web-index/provinces-index-data.js` 内含 `verified_count`；浏览器端 `window.INDEX_PROVINCES[prov].verified_count` 可读取。
- 省份数自检 = 31（`build-web-index.js:54` 会 `exit 1` 否则）。

## P0-3 隐私合规缺失（小程序 + 网页端）

### 问题
- `index-mini` 无任何隐私合规文件：无 `privacy.json`、无《隐私保护指引》配置、无首次授权弹窗（全仓检索 `privacy|隐私保护指引` 无匹配）。微信小程序上线**强制**要求隐私协议，缺失将审核驳回。
- `web-index` 无隐私政策页 / 同意条幅。

### 修复规格
**小程序端（index-mini）**
1. 新建 `index-mini/privacy.json`（微信隐私协议声明文件，放小程序根目录），声明本小程序收集的个人信息类型与用途，例：
   ```json
   {
     "policyVersion": "1.0.0",
     "needAuthorization": true,
     "description": "本小程序仅在本机计算你填写的逐年缴费基数与社平数据，不上传、不收集身份信息与敏感个人信息。",
     "permissionList": [],
     "privacyProtocol": { "url": "https://your-domain/privacy.html" }
   }
   ```
   （具体字段以微信最新《小程序隐私保护指引》规范为准；用微信开发者工具"隐私协议"检测校验。）
2. 在微信公众平台（mp.weixin.qq.com）后台**填写并提交《隐私保护指引》**，并于仓库留底 `index-mini/PRIVACY.md` 说明数据流向（仅本地计算、不收集身份信息）。
3. **首次授权弹窗**：在 `app.js` 或 `pages/index/index.js` 增加隐私同意流——
   - 启动时 `wx.getPrivacySetting` / `wx.requirePrivacyAuthorize` 拉起系统隐私弹窗；
   - 用户点"同意"后方可使用计算；拒绝则提示功能受限。
   - 建议复用官方 `privacy-popup` 组件或自写 modal，文案："我们仅在本机计算你输入的缴费数据，不会上传。点击同意即代表你已知晓《隐私保护指引》。"

**网页端（web-index）**
1. 新建 `web-index/privacy.html`（或站点内 `/privacy` 路由）发布隐私政策，内容与小程序一致。
2. 首次进入计算页展示同意条幅（cookie-style），同意前禁用"开始计算"按钮；同意状态存 localStorage。

### 验证
- 小程序：微信开发者工具"代码质量 / 隐私检测"通过；后台《隐私保护指引》审核通过；冷启动首次弹窗出现且"拒绝"可阻断。
- 网页端：未同意时计算按钮禁用；同意后 localStorage 记忆、可正常计算；隐私页可访问。
