# 缴费指数小程序 · 测试方案（统一表单版）

> 已重构为**统一傻瓜表单**：不再区分 A/B/C 颗粒度、正向/反向模式。
> 用户能填什么信息就填什么，系统自动出对应精度的结果。
> 所有「预期结果」由引擎实跑，与 `scripts/verify-index-mini.js` 对齐。
> 使用前请确认已上传最新版 `calcIndex` 云函数（含统一分派逻辑）。

---

## 表单字段（一个页面填完）

| 字段 | 必填 | 说明 |
|------|------|------|
| 参保省份 | ✅ | 下拉选择 |
| 参加工作时间 | ✅ | 年 + 月 |
| 累计缴费月数 | ✅ | 社保 App 查到的缴费月数 |
| 月均缴费基数 | 选填 | 「每月都差不多」就填这一个；不知道每年变化也填它 |
| 逐年明细 | 选填 | 展开后可填某些年份的基数（JSON），比月均更准 |
| 当前账户余额 | 选填 | 填了自动反推指数，与正向结果互相印证 |

**系统自动决定算什么**：
- 填了基数（月均或逐年）→ 正向：平均指数 + 估算余额 + 逐年明细
- 填了余额 → 反推：推算平均指数
- 都填 → 两者都出，相差大时提示核对

## 用例 1：只填月均基数（最常用）

**填法**：吉林 / 参加 1998年7月 / 累计月数 `295` / 月均缴费基数 `5000` / 余额留空

**预期结果**：
| 项目 | 值 |
|------|------|
| 平均缴费工资指数（正向） | **3.4036** |
| 个人账户余额（估算） | **¥224077.42** |
| 来源标注 | 基于您填的月均基数估算 |

> 注：5000 对吉林（社平较低）是偏高的基数，所以指数 3.4 较高，属正常。填你真实月均基数即可。

## 用例 3：基数 + 余额都填（两者对照）

**填法**：吉林 / 1998年7月 / 295月 / 月均基数 `5000` / 余额 `204822.97`

**预期结果**：同时展示
- 正向指数 **3.4036**
- 反推指数 **1.9888**
- 页面提示「两者若接近说明数据吻合，差异大建议核对」

> 本例两值差较大，是因为示例的「月均5000」与「余额204822」本身不自洽（月均5000 应对应约 22 万余额）。用真实数据填写时两者会接近。

## 用例 4：填逐年明细（最准）

**填法**：吉林 / 参加 2020年1月 / 累计月数 `35` / 展开逐年明细填：
```json
[
  { "year": 2020, "months": 12, "baseAvg": 4980.5 },
  { "year": 2021, "months": 12, "baseAvg": 5450.3 },
  { "year": 2022, "months": 11, "baseAvg": 5890.1 }
]
```

**预期结果**：与只填月均不同，会用逐年真实值
| 项目 | 值 |
|------|------|
| 平均缴费工资指数（正向） | **0.8787** |
| 个人账户余额（估算） | **¥16081.14** |
| 来源标注 | 基于逐年明细计算 |

## 标杆用例：吉林真实样本（295 个月，权威校准）

> 来源：你提供的《个人缴费信息查询.xls》（吉林长春 1998–2022，295 个月真实逐月数据）。
> 数据量大不适合手工填，由 `index-engine/test-calcIndex.js` 自动跑。

**A 详细 + 正向（脚本）**：平均指数 **1.7836** / 账户余额 **¥204822.97** / 1999 年指数 0.647（与人工校准精确匹配）

**C 最简 + 反推（用上面余额，脚本）**：推算指数 **≈1.9888** / 收敛

> 反推 1.9888 ≠ 正向 1.7836 是数学特性（反推假设每年指数恒定，把波动拉平），非 bug。

## 你原截图那组的建议填法

原截图：吉林 / C 最简 / 1998年7月 / 342个月 / 正向 → 全零（旧版）。
新表单下正确用法：
1. **想反推指数**：填「当前余额」= 你社保 App 看到的真实余额（如 69000 或 204822）→ 出推算指数
2. **想正向算**：填「月均缴费基数」= 你大概每月缴的基数（如 5000）→ 出平均指数 + 估算余额
3. **想最准**：展开逐年明细，把每年基数填进去

## 判定标准

- 用例1/4 结果与上表一致 → 正向（月均 / 逐年）正常 ✅
- 用例2 反推有值且收敛 → 反推正常 ✅
- 用例3 同时出两个指数 → 分派逻辑正常 ✅
- 任何用例出现 NaN / 崩溃 / 结果与预期差 >5% → 截图反馈 ❌
