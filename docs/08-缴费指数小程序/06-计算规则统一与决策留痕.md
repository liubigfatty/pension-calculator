# 缴费指数小程序 · 计算规则统一与决策留痕

> 文档目标：把「本人平均缴费指数小程序（全国 31 省版）」的计算口径、边界处理、校准标志等**已拍板规则**集中留痕，作为后续实现、测试、审计的唯一依据。
> 留痕日期：2026-07-09（口径决策）；本文档整理：2026-07-12。
> 状态：**已批准（ratified）**。除非团队 Lead 重新发起评审，否则不复议。

## 0. 总览（五条铁律）

1. 分母口径 = **当年社平·月加权**
2. 单年指数必须**封顶 3.0 / 保底 0.6**
3. 断缴年处理遵循 **GAP_ZERO_PROVINCES**（5 省）
4. 缺失年（无社平 / 未缴费）有明确处置
5. **verified_count** 为省数据校准信心标志，随数据包下发

---

## 规则① 分母口径：当年社平·月加权

- **规则正文**：第 i 年平均缴费指数 aᵢ = 当年月均缴费基数 ÷ 当年全省职工月社平工资（Aᵢ）。平均指数 = Σ(aᵢ × 当年月数) ÷ Σ(当年月数)（按月加权）。
- **决策**：采用「当年社平」口径，金标准平均指数 = **1.6238**（吉林样本 295 月逐月正向）。
- **来源**：`docs/08-缴费指数小程序/02-校准报告.md:33`（2026-07-09 决策，推翻"上年更官方"旧判）；代码实现 `index-mini/cloudfunctions/calcIndex/calcIndex.js:129-130`（取同年社平键）、`:144`（`yearIndex = rec.baseAvg / socialAvg`）。
- **理由**：用户提供的官方公式字面即 aᵢ/Aᵢ = 当年缴费工资额 / 当年全省职工平均工资；原"上年社平更官方"判断已被推翻。主养老金引擎 `engine.js:350-364` 账户积累同样用同年社平，口径一致。

---

## 规则② 封顶保底：单年指数 ∈ [0.6, 3.0]

- **规则正文**：每一年计算出的 aᵢ 必须 clamp 到 [0.6, 3.0]；超出区间按边界计。
- **决策**：`aᵢ_clamped = min(3.0, max(0.6, aᵢ))`。**仅作用于平均指数的分子/分母，不影响个人账户入账**（入账用实际基数 × 8%）。
- **来源**：`index-mini/AGENTS.md:27`（"平均指数一般在 0.6~3.0 之间"）；缺失实现见 P0-1（`calcIndex.js:144` 当前无 clamp）。
- **理由**：缴费工资指数法定上下限为社平 60%~300%；不封顶会让高收入年指数虚高、不保底会让低基数年拉低失真。当前三引擎均未 clamp，属 P0 缺陷（详见 `07-P0修复规格.md` P0-1）。

---

## 规则③ 断缴年（gap year）处理：GAP_ZERO_PROVINCES

- **规则正文**：北京 / 天津 / 陕西 / 浙江 / 云南 5 省，计算平均指数时用"应缴费年限"作分母，断缴年不仅计入分母、分子还按 0 计（即该年 aᵢ = 0），从而严重稀释平均指数；其余省份断缴年直接忽略（不进公式）。
- **决策**：以 `GAP_ZERO_PROVINCES = {beijing, tianjin, shaanxi, zhejiang, yunnan}` 为唯一判定集合；该标志由云函数入口 `index.js:13, :40` 注入 `calculateIndex({ gapYearCountsInAvg })`。
- **来源**：`index-mini/cloudfunctions/calcIndex/index.js:13`（硬编集合）、`:40`（gapYearCountsInAvg 透传）；前端 `pages/index/index.js:64` 重复一份用于展示。
- **理由**：这 5 省人社规则以"应缴年限"为分母，断缴即零指数；与其余省"实际缴费年限"口径不同，属省份级硬规则，不可泛化到其他省。
- **已知代码缺口（H1，非本文档决议，待 P0 之后修）**：`calcIndex.js:101` 的 `isGapYear = firstContribYear != null && year >= firstContribYear` 缺 upper bound，退休后年份可能被误判为断缴；修复规格见后续缺陷单。

## 规则④ 缺失年处理（无社平 / 未缴费）

- **规则正文**：
  - **(a) 某年无社平数据**（socialAvg 缺失 / ≤0）：该年从平均指数公式中**剔除**（不进分子、不进分母），`yearsDetail` 记 `index:null, note:"缺少Y年社平"`。
  - **(b) 用户未缴费年**（前端传 baseAvg=0）：对 GAP_ZERO 省按规则③计入分母（aᵢ=0）；对非 GAP_ZERO 省应识别为断缴并排除（依赖规则③的 `gapYearCountsInAvg` 判定，不能把 baseAvg=0 一律当真实缴费）。
  - **(c) 真实低基数年**（baseAvg 极低但仍 >0）：按实际值计算，再经规则②保底到 0.6。
- **决策**：前端契约——逐年明细数组只发"应覆盖的缴费年"，断缴年 baseAvg 传 0 且由 GAP_ZERO 标志决定其是否计入；引擎侧对 socialAvg 缺失年一律剔除。
- **来源**：`calcIndex.js:132-140`（socialAvg 缺失剔除）、`:144-148`（正常年加权累加）、前端 `pages/index/index.js:125-128`（空年 baseAvg=0 上送）。
- **理由**：剔除无分母年避免 NaN / 失真；区分"真低基数"与"断缴"是平均指数准确性的前提。baseAvg=0 的歧义需在 P0-1 / 规则③落地时一并澄清前端契约。

## 规则⑤ verified_count：省数据校准信心标志

- **规则正文**：`verified_count` = 该省 `avg_salary_history` 中已由人工对照官方来源（人社公布 / 核验表）核对的**年数**。0 或缺失表示尚未核实。
- **决策**：
  - 该字段随社平数据包下发给小程序 / 网页端（`provinces-data.js` / `web-index/provinces-index-data.js`），用于结果页提示"该省社平数据已核对 N 年 / 尚未核对"。
  - 它是**信心标志，不改变计算**；计算永远用 `avg_salary_history` 原始值。
- **来源（真相源）**：`cloudfunctions/calculate/provinces/*.json` 顶层 `verified_count`（例：`henan.json:331`=7、`chongqing.json:4`=0）。当前分布：**13 省 >0**（最高 henan=7）、**7 省 =0**、**11 省字段缺失**（含 jilin）。
- **理由**：全国 31 省社平多为手工录入，用户正在逐省核对；用 `verified_count` 让终端用户一眼判断结果可信度，也便于上线前按"缺失 / 为 0"省份优先补数。当前生成脚本未透传该字段（见 `07-P0修复规格.md` P0-2）。
