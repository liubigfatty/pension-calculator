# 缴费指数小程序 — 实施方案（草案 v0.1）

> 配合 `00-档案-决策与留痕.md` 使用。本方案待 D5/D6 确认后进入实施。

## 一、总体架构
```
[用户]
  └─ 阶段1 独立新小程序(AppID B)  /  阶段2 网页版工具(客服)
       └─ 云函数（单独环境，全部重部署）
            ├─ provinces      （31省社平+利率，复用数据）
            ├─ calcIndex      （正向：明细→指数+余额对账）
            ├─ calcIndexReverse（反向：余额→等效指数）
            └─ （若收费）createOrder / payCallback（复用代码，新OfferID）
```

## 二、数据层
- 复用 provinces 配置（`cloudfunctions/calculate/provinces/*.js` 的 `getEngineConfig()`，31 省全口径社平 + 记账利率）。
- 新环境部署：`provinces` 云函数 + 新 `calcIndex` / `calcIndexReverse` 云函数。
- 按"全云函数"决策，前端调云函数取数（非前端内置）。

## 三、计算引擎（新建 index-engine，复用 provinces 数据）
### 正向 calcIndex(input)
```
input: {
  province, city(双基数省填),
  参加工作时间(YYYYMM), 视同年限(年),
  缴费明细: [{ year, monthlyBase | annualBase }],
  当前个人账户余额(可选, 用于对账)
}
output: {
  每年: { year, base, socialAvg, index },
  平均缴费工资指数,
  复算个人账户余额, 与输入余额对账差值
}
```
### 反向 calcIndexReverse(input)
```
input: { province, 参加工作时间, 累计缴费月数, 当前余额, (可选部分年基数) }
output: 等效平均缴费工资指数
```

## 四、UI/UX（三模式录入，对应三颗粒度）
- **模式A 导入明细**：粘贴/上传 xls 或逐月、逐年填 → 自动按年聚合 → 每年指数 + 余额对账。
- **模式B 关键年**：填参加时间 + 月数 + 余额 + 部分年份基数 → 已知年精确，未知年等效。
- **模式C 最简**：填参加时间 + 月数 + 余额 → 直接出等效平均指数。
- **结果页**：平均指数（大字）+ 每年指数表 + 余额对账（复算 vs 社保网）+ （反向）等效指数说明。
