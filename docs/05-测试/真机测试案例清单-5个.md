工作月份：11

---

```

**预期结果**：
```

个人账户养老金： 596.73 元

## 案例 5：广东-深圳市档位（分档省份）

**测试目的**：验证分档省份的城市选择功能

**输入参数**：
```
省份：广东
城市类型：深圳市  ← 选这个，因为深圳基数和全省不同
性别：女
退休类型：工人 50岁
出生年份：1975
出生月份：8
工作年份：1995
工作月份：5
退休年份：2025
退休月份：8
---
平均缴费指数：0.7
个人账户余额：100000
实际缴费年限：25
视同缴费年限：3
```

**预期结果**：
- 小程序应该显示**深圳的计发基数**（不是全省基数）
- 计算结果应该和用全省基数算出来的不同

**判断标准**：
1. 能正常选择"深圳市"档位 ✅
2. 结果页显示的计发基数 = 深圳的基数 ✅
3. 计算结果合理，不Crash ✅
