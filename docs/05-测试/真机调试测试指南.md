# 真机调试测试指南

**版本**：v1.0
**适用人员**：测试人员/产品经理
**预计耗时**：30-45 分钟

---

## 一、准备工作

### 1.1 打开微信开发者工具

1. 启动**微信开发者工具**
2. 导入项目：`C:\Users\14041\AppData\Roaming\winclaw\.openclaw\workspace\养老金计算平台`
3. 确认 AppID 已配置（如需测试云函数，需配置真实 AppID）

### 1.2 启动真机调试

1. 在开发者工具工具栏，点击 **「真机调试」** 按钮
2. 会生成一个**二维码**
3. 用**手机微信**扫描二维码
4. 手机上会打开小程序，并进入调试模式

> ⚠️ 手机和电脑需要在同一 Wi-Fi 网络下，否则可能无法连接

---

## 二、测试流程（按优先级）

### 🔴 P0 核心流程测试

---

### 测试点 1：首页加载

**操作**：
1. 手机微信扫描二维码，打开小程序
2. 观察首页是否正常显示

**检查项**：
- [ ] 页面正常加载，无白屏
- [ ] 标题显示"养老金测算"
- [ ] 有"开始计算"按钮

**如遇问题**，记录：
- 现象：___________
- 手机型号：___________
- 微信版本：___________

---

### 测试点 2：省份选择

**操作**：
1. 点击"开始计算"
2. 进入省份选择页
3. 选择一个省份（如"北京"）
4. 点击"下一步"

**检查项**：
- [ ] 省份列表正常显示（31 个省）
- [ ] 选择省份后，能正常跳转下一步
- [ ] 如有分档省份（广东/湖北等），能选择城市类型

**测试数据建议**：
- 北京（无分档）
- 广东（有分档：广州/深圳）
- 浙江（双指数省份）

---

### 测试点 3：个人信息输入

**操作**：
1. 在 step2 页面，填写个人信息
2. 点击"下一步"

**测试数据（北京案例 1）**：
```
性别：女
退休类型：工人 50 岁
出生年份：1973
出生月份：10
工作年份：1995
工作月份：11
退休年份：2023
退休月份：10
```

**检查项**：
- [ ] 输入框正常显示，可输入
- [ ] 日期选择器正常工作
- [ ] 点击"下一步"能正常跳转

---

### 测试点 4：缴费信息输入 + 计算

**操作**：
1. 在 step3 页面，填写缴费信息
2. 点击"计算"按钮

**测试数据（北京案例 1）**：
```
平均缴费指数：0.4584
个人账户余额：116362.74
实际缴费年限：21
视同缴费年限：4.01
1997年底前缴费月数：2.58 年
```

**检查项**：
- [ ] 点击"计算"后，显示 loading 状态
- [ ] 计算完成后，跳转到结果页
- [ ] 结果页显示：
  - 基础养老金
  - 个人账户养老金
  - 过渡性养老金
  - 总额

**结果对比（北京案例 1 预期值）**：
```
基础养老金：2128.43 元
个人账户养老金：570.50 元
过渡性养老金：151.00 元
总额：2849.93 元
```

> 实际计算结果与预期值误差 < 1% 为通过

---

### 测试点 5：结果页展示

**操作**：
1. 查看结果页完整内容
2. 尝试点击"重新计算"
3. 尝试分享功能

**检查项**：
- [ ] 结果页数据展示完整、清晰
- [ ] 点击"重新计算"能返回输入页
- [ ] 分享功能正常（右上角"...->分享给朋友"）

---

### 🟡 P1 边界情况测试

---

### 测试点 6：参数校验

**操作**：
1. 在 step2 或 step3 页面，**故意留空必填字段**
2. 点击"下一步"或"计算"

**检查项**：
- [ ] 显示友好的错误提示（如"请填写退休年份"）
- [ ] 不Crash，不白屏

---

### 测试点 7：隐私协议弹窗

**操作**：
1. 在手机微信中，**删除小程序**（长按小程序图标->删除）
2. 重新扫描二维码，首次进入小程序

**检查项**：
- [ ] 首次进入时，弹出隐私协议弹窗
- [ ] 有"同意"和"拒绝"按钮
- [ ] 点击"同意"后，弹窗关闭，可正常使用
- [ ] 点击"拒绝"后，弹窗关闭，功能受限或提示

---
