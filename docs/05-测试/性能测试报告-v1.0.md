## 一、性能指标测量

### 1.1 启动性能

| 指标 | 目标值 | 测量方式 | 实际值 | 是否达标 |
|------|--------|---------|--------|---------|
| 冷启动时间 | < 1.5秒 | 开发者工具→详情→启动性能 | 待测 | ☐ |
| 热启动时间 | < 0.5秒 | 开发者工具→详情→启动性能 | 待测 | ☐ |
| 首屏渲染时间 | < 1.0秒 | Performance→Rendering | 待测 | ☐ |

**测量步骤**：
1. 打开微信开发者工具
2. 点击「详情」→「启动性能」
3. 记录冷启动、热启动时间
4. 使用 Performance 标签记录首屏渲染时间

---

### 1.2 包大小分析

| 指标 | 目标值 | 实际值 | 是否达标 |
|------|--------|--------|---------|
| 主包大小 | < 2MB | 待测 | ☐ |
| 总包大小（含分包） | < 20MB | 待测 | ☐ |

**测量方式**：开发者工具→详情→基本信息→「代码依赖分析」

**优化措施（如遇不达标）**：
- [ ] 图片资源压缩（使用 TinyPNG 或 CDN WebP）
- [ ] 将非核心页面移入分包（`subpackages/`）
- [ ] 删除未使用的组件和库
- [ ] 开启开发者工具「代码压缩」选项

---

### 1.3 渲染性能

| 指标 | 目标值 | 实际值 | 是否达标 |
|------|--------|--------|---------|
| setData 单次耗时 | < 50ms | 待测 | ☐ |
| setData  payload 大小 | < 256KB | 待测 | ☐ |
| 页面切换耗时 | < 300ms | 待测 | ☐ |

**测量方式**：开发者工具→调试器→Performance → 录制操作过程

**优化措施（如遇不达标）**：
- [ ] 减少 setData 调用频率（合并多次更新）
- [ ] 使用 `pureDataPattern` 标记非渲染数据
- [ ] 长列表使用虚拟列表（`recycle-view`）
- [ ] 避免在 `onPageScroll` 中频繁 setData

---
