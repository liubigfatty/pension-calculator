## 二、setData 使用检查

### 检查结果

| 页面 | setData 调用次数 | 评估 | 建议 |
|------|----------------|------|------|
| step1.js | 3次 | ✅ 合理 | 无需优化 |
| step2.js | 10次 | ⚠️ 偏多 | 可合并部分更新 |
| step3.js | 15次 | ⚠️ 偏多 | 计算时频繁更新loading状态 |
| result.js | 3次 | ✅ 合理 | 无需优化 |
| index.js | 2次 | ✅ 合理 | 无需优化 |

### 问题分析

**step3.js 问题**：
- 计算过程中多次调用 `setData({ estimating: true/false })`
- 每次 `setData` 都会跨 JS-Native 桥，影响性能

**优化建议**（可选，非紧急）：
```javascript
// 现在：频繁更新 loading 状态
this.setData({ estimating: true })
// ... 计算 ...
this.setData({ estimating: false })

// 建议：用 wx.showLoading/hideLoading 替代
wx.showLoading({ title: '计算中...' })
// ... 计算 ...
wx.hideLoading()
```

### 结论

当前 `setData` 使用**不影响功能**，但**有优化空间**。
- 如果计算时间 < 1秒，无需优化
- 如果计算时间 > 2秒，建议按上述方式优化
