# 福建企业职工养老金计算规则核对（2026-07-23）

## 来源
用户提供闽政〔1996〕13号体系下的福建企业养老金计算办法（含完整实例解析）。本次仅做核对归档，未改引擎代码（用户指令：总结 / 归档 / 推送 / 下一省）。

## 福建三大亮点规则

### 规则1：缴费年限按半段进整（不足半年按半年，大于半年不足一年按一年）
- 适用范围：基础养老金用的「累计缴费年限」+ 过渡性养老金用的「建账前缴费年限」。
- 进整规则：余月数 ≤ 6 → 进到 0.5 年；余月数 > 6 → 进到 1 年。
- 实例：38年2个月→38.5年；10年4个月→10.5年；27年10个月→28年；22年10个月→23年。

### 规则2：建账前平均缴费指数（β）低于 0.75 按 0.75 计
- β = 建账前平均缴费指数；β ≤ 0.75 时，建账前部分按 0.75 参与加权。

### 规则3：本人平均缴费指数（α）分段确认、全程合并计算
- 1990年底前缴费年限指数按 1.0；
- 1991–1995年（建账前）按各年实际缴费指数；
- 建账后（1996–退休当年）按实际，退休当年在岗职工平均工资按上年标准；
- 视同缴费年限指数按 1；未缴费年限不参与。
- 公式：
  - β = [1×m + Σ(gᵢ/sᵢ)] / (m + i)
  - α = [min(β,0.75)×(m+i) + Σ(xⱼ/cⱼ)] / (m + i + n)
- 实例：β = 1.021，α = 0.954。

## 引擎现状核对
- **规则1（半段进整）**：引擎 `engine/pension-engine.js` L1714-1727 已有 `rounding.years_half_step` 实现，注释明确写「福建等省份：年限按半段进整」。但 **`fujian.js` 的 `getEngineConfig()` 未启用 `rounding`**，故当前福建计算并未进整。→ 待启用 `rounding: { years_half_step: true, index_decimal: 4 }`。
- **规则2+3（分段 α/β 与 0.75 封底）**：引擎 `avgIndex` 为前端用户直接输入的标量（`web/app.js:179`），不摄入逐年缴费工资，故 **无法在引擎内自动计算 β/α 及其 0.75 封底**。福建实质是「双指数」省份：基础养老金用 α、过渡性养老金用 β（封底 0.75）。→ 待建：福建 α/β 逐年工资计算器（前端）+ 引擎双指数分支；或在文档/前端提示用户按此法手算 α 后填入。
- **过渡性养老金年限**：福建过渡性养老金用「建账前缴费年限（含视同+建账前实际）」，当前 `usePreAccountYears:false` 时退回仅用 `sightYears`（视同），会少算 1991–1995 建账前实际年限。→ 待启用 `usePreAccountYears: true`（preEnd = account_start = 1996-01）。

## 结论
福建三条规则中，规则1的引擎实现已就绪、仅待启用；规则2/3 涉及分段指数，需前端逐年工资采集 + 引擎双指数支持，属后续功能。本次只核对归档。
