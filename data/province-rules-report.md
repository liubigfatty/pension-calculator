## 吉林省

### 1. 养老金组成部分

根据 `provinces/jilin.json` 的 `modules` 配置：

| 组成部分 | 是否启用 | 公式 |
|---------|---------|------|
| **基础养老金** (`basic_pension`) | ✅ 启用 | `(退休地计发基数 + 全省计发基数 × 指数) / 2 × 累计缴费年限 × 1%` |
| **基础养老金增发部分** (`extra_pension`) | ✅ 启用 | 实际缴费超20年部分分段累加计算 |
| **个人账户养老金** (`personal_account`) | ✅ 启用 | `个人账户累计储存额 ÷ 计发月数` |
| **过渡性养老金** (`transitional_pension`) | ✅ 启用 | `全省计发基数 × 视同缴费年限 × 指数 × 系数(>20年:1.4%,≤20年:1.2%)` |
| **其它加发** (`special_addition`) | ❌ 禁用 | — |

### 2. 关键参数

| 参数 | 数值 | 说明 |
|------|------|------|
| 建账时间 | 1995年7月 | — |
| 视同缴费起始 | 1995年7月 | — |
| 过渡系数（>20年） | 1.4% | 累计缴费年限超过20年 |
| 过渡系数（≤20年） | 1.2% | 累计缴费年限不超过20年 |

### 4. 基础养老金增发部分（extra_pension）详细规则

**触发条件**：实际缴费年限超过20年

**分段计算**：

| 实际缴费年限 | 增发比例 |
|-------------|---------|
| 21-25年 | 0.15%/年 |
| 26-30年 | 0.20%/年 |
| 31年及以上 | 0.25%/年 |

**公式**：
```
增发金额 = 指数化月平均缴费工资 × (分段增发比例之和)
```

### 4. 增发养老金（extra_pension）详细规则

**政策依据**：川劳社发〔2006〕18号

```

```

**增发比例**（因人而异）：
- 独生子女：0.1%
- 劳模：0.1%-0.3%
- 高级职称：0.1%
- 合计比例：0.1%~0.4%+

- 案例4：增发247.64元（2024年退休，比例0.1%）
- 案例5：增发869.95元（2024年退休，比例0.4%）
- 案例6：增发317.88元（2025年退休，比例0.1%）

## 统一引擎配置规范

### 1. MODULES 标准值

| 模块Key | 模块名称 | 适用省份 | 说明 |
|---------|---------|---------|------|
| `base` | 基础养老金 | 所有省 | 必需模块 |
| `extra` | 增发养老金 | 吉林、四川 | 各省政策不同 |
| `personal` | 个人账户养老金 | 所有省 | 必需模块 |
| `transition` | 过渡性养老金 | 所有省（有视同缴费的） | 系数各省不同 |
| `other` | 其它加发 | 黑龙江、甘肃、云南 | 各省内容不同 |

### 2. 各省MODULES配置总览

| 省份 | MODULES | 说明 |
|------|---------|------|
| **吉林省** | `['base', 'extra', 'personal', 'transition']` | `extra`=基础养老金增发部分（长缴费奖励） |
| **黑龙江省** | `['base', 'personal', 'transition', 'other']` | `other`=御寒津贴/劳模补贴 |
| **甘肃省** | `['base', 'personal', 'transition', 'other']` | `other`=调节金 |
| **云南省** | `['base', 'personal', 'transition', 'other']` | `other`=独生子女补贴（全省退休人员人均养老金×5%） |
| **上海市** | `['base', 'personal', 'transition']` | 无增发、无其它加发 |
| **北京市** | `['base', 'personal', 'transition']` | 过渡性养老金=G同+G实 |
| **江苏省** | `['base', 'personal', 'transition']` | 过渡性养老金系数1.2% |
| **四川省** | `['base', 'extra', 'personal', 'transition']` | `extra`=增发养老金（针对1995年底前未建账缴费年限） |

### 3. 配置一致性检查规则

**规则1**：JSON `modules.*.enabled` 必须与 JS `MODULES` 一致

- 如果 `modules.extra_pension.enabled = true`，则 `MODULES` 必须包含 `'extra'`
- 如果 `modules.special_addition.enabled = true`，则 `MODULES` 必须包含 `'other'`
- 如果 `modules.adjustment_fund.enabled = true`，则 `MODULES` 必须包含 `'other'`（甘肃调节金）

**规则2**：`MODULE_LABELS` 必须与 `MODULES` 一一对应

**规则3**：`MODULE_FORMULAS` 必须与 `MODULES` 一一对应

**规则4**：`MODULE_COLORS` 必须与 `MODULES` 一一对应
