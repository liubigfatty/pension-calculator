# 养老金测算 SKILL

## 功能
根据用户的参保信息，测算：
- **预计退休时间**（含延迟退休政策自动计算）与**退休年龄**。
- **每月养老金**：基础养老金 + 个人账户养老金 + 过渡性养老金 + 各省加发项。
- 覆盖全国 31 省，区分企业职工 / 灵活就业、男 / 女不同法定退休年龄。

## 何时调用
- 用户问「我能领多少养老金」「我退休金大概多少」「我什么时候能退休」「帮我算下养老金」。
- 用户提供了参保地、人员类型、出生年月、参加工作年月、平均缴费指数（或愿意补充）。

## 输入
原子接口 `calculatePension`：
- `province`（必填）：参保地，支持中文省名（「吉林省」「浙江」「北京」）或拼音（jilin/zhejiang/beijing）。
- 人员类型（二选一填法）：
  - `personType`：企业男 / 企业女50 / 企业女55 / 灵活男 / 灵活女（推荐，一个字段搞定）；
  - 或分开填 `gender`（男/女）+ `identity`（企业职工/灵活就业）+ `femaleRetireType`（50/55，仅企业女职工）。
- `birthDate`（必填）：出生年月，如「1975-02」。
- `workStartDate`（必填）：参加工作/首次参保年月，如「1998-07」。
- `averageIndex`（必填）：本人平均缴费工资指数，如 0.6 / 1.0 / 3.0（灵活就业按档次，60%=0.6）。
- `personalAccount`（可选）：个人账户余额（元），留空时引擎自动估算。
- `retirePlan`（可选）：normal（正常退休）/ early（弹性提前退休），默认 normal。

## 输出（结构化）
- `retireDate`：预计退休年月
- `retireAge`：退休年龄（如「62岁7个月」）
- `monthlyPension`：每月养老金（元）
- `province` / `averageIndex`

## 示例
用户：「我是吉林的企业职工男，1975 年 2 月出生，1998 年 7 月参加工作，平均缴费指数 1.0，能领多少？」
→ province="吉林省"，personType="企业男"，birthDate="1975-02"，workStartDate="1998-07"，averageIndex=1.0，调用 calculatePension。

## 数据口径
- 社平工资 / 计发基数：各省市人社部门公布的官方数据。
- 记账利率：1996–2025 年统一利率表。
- 退休年龄：已计入国家渐进式延迟退休政策。
- 结果为测算值，仅供参考，不作为社保待遇核定依据。

## 结果呈现（账号卡片 → 接力小程序）
- 计算完成后，结果以「账号卡片」呈现核心结论（退休时间、退休年龄、每月养老金）。
- 用户点击卡片可接力进入小程序结果页（`/pages/result/result`），查看基础/个人账户/过渡性养老金等完整明细。
- 话术上先给结论，再引导「点击卡片进入小程序查看完整明细」。

## 边界
- 需要参保地 + 人员类型 + 出生年月 + 参加工作年月 + 平均缴费指数五要素才能测算；缺失请追问。
- 不提供理财、收益预测、投资建议。

## 示例对话（意图识别 few-shot）
**① 完整信息一次给全**
用户：「浙江灵活就业女，1978年5月生，2005年3月开始交，指数0.8，退休金多少」
→ calculatePension：province="浙江"，personType="灵活女"，birthDate="1978-05"，workStartDate="2005-03"，averageIndex=0.8。

**② 信息不全需追问**
用户：「帮我算下养老金」
→ 先问齐：参保地？人员类型（企业/灵活、男/女）？出生年月？参加工作年月？平均缴费指数？再调用。

**③ 只给档次百分比**
用户：「北京灵活就业男按60%交的」
→ averageIndex=0.6（60%档=0.6），继续追问出生年月与首次参保年月。

**意图关键词速查**：
- 「能领多少 / 退休金多少 / 养老金多少 / 什么时候退休」→ `calculatePension`
- 缺任一要素（参保地/人员类型/出生年月/参保年月/指数）→ 先追问再计算。
